# 作者风格校准

## 使用时机

当用户要求"像我写的""模仿某个作者/账号的风格""根据样本改写"时，读取本文件。目标是提取可复用的**表达习惯与结构偏好**，而不是复制独特句子、冒充具体身份或伪造经历。

## 样本要求

理想样本为 3–7 篇同一作者、同一语域或相近任务的文本。每篇最好超过 300 字。样本不足时，只做轻量校准，不声称完全复现。若用户提供的是受版权保护的在世作者文本，应只提取高层写作特征，不仿写可识别表达。

## 风格指纹表

| 维度 | 观察问题 | 记录方式 |
|---|---|---|
| 段落长度 | 作者偏短段、厚段还是混合 | 平均段长、短段比例、最长段功能 |
| 句式复杂度 | 长句多还是短句多 | 长句承担解释、判断、铺垫还是情绪 |
| 开头方式 | 直接结论、场景、问题、事实、反常识 | 记录 2–3 种高频开头模式 |
| 转折方式 | 但是、不过、问题是、真正麻烦的是 | 区分书面转折与口语转折 |
| 标点习惯 | 冒号、分号、括号、引号、破折号 | 统计频率和典型用途 |
| 人称使用 | 我、我们、你、大家是否常见 | 记录出现位置和语气功能 |
| 事实锚点 | 是否常用时间、地点、数据、专名 | 保留原文事实，不补不存在来源 |
| 情绪表达 | 直接情绪词还是通过判断呈现 | 禁止用"令人深思"替代真实判断 |
| 收尾方式 | 结论、问题、行动、留白 | 避免万能升华 |

## 校准流程

1. **先读样本，不急着改写**。提取段落谱系、句式节奏、标点、开头和收尾方式。
2. **生成内部语域模板**。用 8–12 条短规则记录，不向用户展示，除非用户要求。
3. **区分可迁移与不可迁移**。可迁移的是节奏、语气、结构和表达密度；不可迁移的是真实经历、专属身份、未授权素材和独特金句。
4. **先按用户任务写出保真稿**。不要一开始就过度模仿。
5. **再做风格贴合**。调整段落厚度、转折、开头、收尾和语气强度。
6. **最后做 AI 痕迹补丁**。避免样本风格被模型化地平均掉。

## 内置语域模板：小红书短文（AI 产品评论语域）

从24张写作样本中提取，适用于 AI 产品分析、职场洞察、产品案例等语域的改写任务。调用时无需用户重新提供样本。

```
作者指纹：小红书短文
- 语域：小红书评论随笔，面向年轻从业者，介于口语和专栏之间
- 段落：极短段为主，1-3句一段；用数字（1、2、3）做章节分隔，不用"第一部分"
- 句式：铺垫用长句，结论用短句，节奏是"展开→收紧"；动词驱动，不用名词堆叠
- 开头：直接进入场景或事实，不预告结构；偶尔用设问引入
- 转折：用"但""而""不是……而是……"，口语化；不用"然而""尽管如此"
- 标点：冒号用于引出列举，分号用于并列判断，不滥用破折号
- 人称："你"对读者，"我"用于自己判断，"我们/咱们"拉近距离
- 情绪：不用情绪词，用判断和事实传递情绪；允许有立场，不强行平衡
- 事实锚点：喜欢用具体公司名、产品名、时间跨度锚定判断；不用"业内人士"
- 收尾：以具体判断或行动收尾，不升华；不用"希望对你有帮助"
- 禁止项：首先/其次/此外/综上所述、不仅……更是……、挑战与机遇并存、令人深思/值得期待、加粗整句论点、每段都有小标题
```

## 内置语域模板：小红书科技资讯博主（成熟内容号）

适用于 AI 资讯、科技公司动态、产品发布、行业观察和中文社媒短评。调用时无需用户重新提供样本，但必须保留原文事实和不确定性，不伪造现场经历、朋友爆料或内部消息。

```markdown
作者指纹：小红书科技资讯博主（成熟内容号）
- 语域：中文科技资讯短评，介于小红书笔记、公众号短评和长微博之间；不是种草口吻，也不是新闻通稿
- 段落：中等厚度为主，2–4句一段；每段要有事实锚点、对象动作或明确判断，少用纯口号短段
- 句式：中长句承载信息，短句用于落判断；中文语序优先，如“我感觉最强烈的是”，不用“我最强的感觉是”
- 开头：从最大感触、最反常的事实或最值得聊的判断进入；优先用“今天不用急着记住每个发布点”“我感觉最强烈的是……”“更值得盯的是……”“这条消息最有意思的地方在于……”这类口语入口，先抓重点，再补信息
- 转折：用“先说说”“反观”“再聊”“问题在这儿”“更关键的是”“这事有意思在于”；不用“首先、其次、此外、综上”
- 标点：逗号和句号为主，冒号只在真正引出解释或清单时使用；不用连续感叹号，不靠表情符号补情绪
- 人称：可以弱第一人称表达观察，如“我看下来”“我更关心”“我会先观望”；不得补“我在现场”“我朋友说”等未提供经历
- 情绪：克制但有立场，允许轻微吐槽、疑虑和偏好；不用“令人震撼、意义重大、值得期待”替代具体判断
- 事实锚点：保留公司名、产品名、时间、数字、发布渠道、消息来源和不确定性；传闻必须写“据报道、可能、尚未确认、目前看”
- 收尾：停在具体判断、限制条件、后续变量或一个问题上；需要总结时用“更值得记住的是……”“真正要看的还是……”“这才是下一轮更现实的考题”这类口语结构，不写“我会记成一句话”“总结来看”，也不升华到时代趋势和宏大未来
- 禁止项：碎片化单句堆叠、新闻通稿零度情绪、强行金句、刻意总结句、底层逻辑/赋能/闭环/生态、未来已来/格局打开/具有重要意义
```

## 中英文句式根本差异（LLM 的系统性偏差）

### 问题来源

LLM 的训练语料以英文为主，生成中文时底层句式逻辑仍是英文的。这不是翻译腔，而是模型的"母语"渗透——英文惯用抽象名词做主语、系动词做谓语、名词短语做宾语；中文母语者惯用人或具体事物做主语、动词做谓语、句子有动作。

这个偏差在所有 LLM 生成的中文里都存在，不是个别词的问题，是句式逻辑的问题。L1 改写时必须专门处理。

### 典型对照

| AI 生成（英文句式渗透） | 中文母语者的写法 |
|---|---|
| 这个产品的起点是一个情绪时刻 | 这个产品从我身边捕捉到的一个情绪中诞生 |
| 核心价值在于激发用户的主动感 | 用完之后用户会觉得"我能说" |
| 这个洞察的关键结论是…… | 这个洞察告诉我…… |
| 评估的维度不是模型自己的评分 | 我不看模型给的分，我看…… |
| 产品的定位转变是必要的 | 我把产品的定位改了 |
| 这一决策的背后逻辑是…… | 我这么做，是因为…… |
| 这个问题的本质是…… | 说到底，这件事是…… |
| 用户的核心诉求在于…… | 用户真正想要的是…… |

### 修复规则

扫描全文中"X是Y"句式，判断是否在描述动作或状态（而非真正下定义）。如果是，把主语改成人或具体事物，谓语改成动词。把抽象名词做主语的句子，改成"谁做了什么"或"我发现……"的结构。连续的名词短语堆叠，拆成有动作的句子。

**判断标准**：这个句子，如果一个中文母语者在聊天里说出来，会不会觉得别扭？别扭的，改；不别扭的，保留。

## 复现边界

不要承诺"完全像本人"。不要复制样本中连续的独特表达。不要把用户没有提供的经历、身份、关系和判断写进正文。若任务需要第一人称，但原文没有足够材料，可以使用弱第一人称，如"我更倾向于把它理解为……"；涉及真实经历时必须询问用户或改为非个人化表达。

## 样本不足时的轻量校准

若只有一段样本或少量聊天记录，只提取明显特征，例如句长、称呼、正式程度和标点习惯。输出时不要过度扩展作者画像。可以提示用户："如果你想更像自己的长期风格，可以再给 3 篇样文。"

## 不同任务的校准强度

| 任务 | 校准强度 | 说明 |
|---|---:|---|
| 邮件、商务回复 | 低到中 | 清楚和边界优先，少模仿口癖 |
| 公众号、专栏 | 中到高 | 段落、判断和收尾最重要 |
| 社媒、朋友圈 | 高 | 语气、停顿、口头习惯很关键 |
| 学术、报告 | 低 | 术语与准确性优先，不做强个性化 |
| 口播 | 中到高 | 句长、停顿和重复方式最重要 |
