# 中文语域与平台风格

## 使用时机

当用户要求改成某个平台、某类读者或某种语气时，读取本文件。语域不是“更口语”或“更高级”的选择，而是文本与场景之间的匹配关系。

## 语域选择表

| 语域 | 适合场景 | 句式与段落 | 应避免 |
|---|---|---|---|
| 正式说明 | 产品说明、政策解读、内部文档 | 中等句长，概念清楚，少情绪 | 公众号式煽动、口语插科打诨 |
| 商业沟通 | 邮件、方案、汇报、客户回复 | 直接说明背景、判断、请求和下一步 | 过度客套、宏大愿景、空泛感谢 |
| 评论随笔 | 公众号、专栏、长微博 | 有判断，有铺垫，段落可长短不齐 | 三段式论证、鸡汤结尾 |
| 知乎回答 | 经验解释、观点拆解、案例说明 | 开头直接回应问题，允许个人判断 | 过多标题、像百科词条 |
| 社媒短文 | 朋友圈、小红书、微博 | 短段落，具体场景，少解释；允许轻微跳跃和偏向 | 教科书口吻、强行金句、每条都解释到满 |
| 熟人聊天 | 私信、群聊、轻量建议 | 短句和省略更自然，允许停顿 | 完整论文式段落、过度礼貌 |
| 口播脚本 | 短视频、播客、直播讲稿 | 口语化承接，句子可被一口气说完 | 长定语、括号、复杂并列 |
| 学术克制 | 摘要、论文润色、研究笔记 | 术语准确，因果谨慎，少修辞 | 情绪化、擅自强化结论 |

## 各语域操作要点

### 正式说明

保持清楚、稳定和可追溯。不要为了去 AI 味删除必要的结构。可以减少“本文将、主要包括、值得注意的是”，但保留必要定义、条件和步骤。句子要有人和动作，不要全是抽象名词。

### 商业沟通

先说结论，再说原因和需要对方做什么。中文商业邮件的真人感来自**明确的责任边界和下一步动作**，不是寒暄。少用“非常荣幸、携手共创、赋能增长、生态闭环”等词。

### 评论随笔

允许作者有判断和停顿。开头可以从一个具体事实、矛盾、疑问或个人观察进入，不要用“在当今快速发展的时代”。段落可以有厚度，结尾不必升华，可以停在一个现实问题上。

### 知乎回答

开头要回应问题本身。可以承认条件和例外，但不要把所有问题都写成“一方面另一方面”。适度使用列表可以提高可读性，但每个小标题都要有具体信息，不能只是“背景、原因、影响、建议”。

### 社媒短文

少解释，多画面。不要把短文写成“观点 + 分析 + 总结”。可以保留不完整句、轻微跳跃和真实情绪，但不要堆叠网络热词。小红书资讯类内容尤其要避免“一条新闻 + 一段意义”的工整循环；可以先写自己最在意的点，再把新闻事实补上。用户没有提供个人经历时，不要编“我今天遇到”，可以写“我看下来”“我更在意”“这条我先观望”。

#### 小红书科技资讯博主（成熟内容号）

这个细分语域适合 AI 资讯、科技产品、商业动态、行业观察等内容。它不是碎片化种草笔记，也不是新闻通稿，而是**有信息密度、有个人判断、有一点修辞感的中文内容号**。开头优先用“今天不用急着记住每个发布点”“我感觉最强烈的是”“更值得盯的是”“这条消息最有意思的地方在于”这类自然入口，先抓住最值得说的点，再补事实。

| 维度 | 写法 |
|---|---|
| 段落形态 | 2–4 句一段为主，不把每个判断都拆成单句；每段至少有一个事实锚点或明确判断 |
| 用词标准 | 可以用“眼花缭乱、稳稳攥住、褪去滤镜、压到台前”这类有画面感的中文，但避开“底层逻辑、赋能、闭环、生态、颠覆”等烂大词 |
| 情绪控制 | 立场要看得出来，但语气保持克制；可以写“我更关心”“这点挺微妙”“我会先观望”，不要全程中立或连续惊叹 |
| 转场方式 | 用“先说说”“反观”“再聊”“问题在这儿”“更关键的是”“这事有意思在于”承接，不用“首先、其次、此外、综上” |
| 事实边界 | 未确认消息保留“据报道、可能、目前看、尚未确认”；不要为了显得有态度把传闻写成定论 |
| 结尾规则 | 开头和结尾这类最需要总结的位置，优先使用口语化结构；可以写“更值得记住的是……”“真正要看的还是……”“这才是下一轮更现实的考题”，停在具体判断、限制条件、下一步变量或没解完的问题上，不升华到“时代、格局、未来已来”，也少用“我会记成一句话”这类刻意总结 |

### 熟人聊天

句子可以不完整，语气要像对具体的人说话。可以用“这个我建议你先别急”“大概是这个意思”，但不要过度使用“哈哈、真的、说实话”。聊天语域最怕模型式耐心：每件事都解释三遍。

### 口播脚本

每句话都要能顺口读出来。长句拆开，书面词换成日常表达。过渡要靠说话节奏，不靠“第一、第二、第三”。重要信息可以重复一次，但不要重复抽象口号。

### 学术克制

保留术语、限定、引用和因果强度。不要把学术文本改成公众号。去 AI 味的重点是减少“具有重要意义、提供了新的视角、未来研究可以进一步探索”等空泛句，把贡献、方法和局限写具体。

## 语域冲突修正

| 冲突 | 表现 | 修正 |
|---|---|---|
| 邮件像报告 | 背景过长，标题过多 | 缩短背景，明确请求和截止时间 |
| 评论像百科 | 定义太多，立场太少 | 增加判断句，删掉常识铺垫 |
| 口播像论文 | 长定语、被动句、括号 | 拆句，换成说话顺序 |
| 学术像营销 | “颠覆、重塑、赋能” | 改成可验证的研究结论 |
| 聊天像客服 | “希望以上信息对你有所帮助” | 换成具体建议或自然停顿 |

## 默认语域决策

如果用户只说“去 AI 味”，先保留原文语域，不主动改平台。如果原文没有明显语域，长文默认“正式说明 + 轻评论”，短文默认“社媒短文”，邮件默认“商业沟通”，问答默认“知乎回答”。
